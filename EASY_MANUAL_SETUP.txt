═══════════════════════════════════════════════════════════════════════
  ШАГ 1: СОЗДАЁМ РЕПОЗИТОРИЙ НА GITHUB (2 минуты)
═══════════════════════════════════════════════════════════════════════

1. Откройте в браузере: https://github.com/new

2. Заполните:
   Repository name: hardcore-messenger
   Description: HardcoreMessenger with HARD+ features
   
3. Выберите: ⚫ Public

4. НЕ ставьте никакие галочки!

5. Нажмите: Create repository

6. Вы увидите пустой репозиторий - ЭТО НОРМАЛЬНО!


═══════════════════════════════════════════════════════════════════════
  ШАГ 2: СОЗДАЁМ ZIP АРХИВ С КОДОМ (1 минута)
═══════════════════════════════════════════════════════════════════════

1. Откройте папку с проектом:
   HardcoreMessenger_ONLINE

2. Зайдите в папки и УДАЛИТЕ (если есть):
   - .git (скрытая папка)
   - Client\bin
   - Client\obj
   - Server\bin
   - Server\obj

3. Выделите ВСЕ папки и файлы:
   - Client
   - Server
   - Shared
   - Все .bat файлы
   - Все .md и .txt файлы
   - Dockerfile
   - railway.json

4. ПРАВОЙ кнопкой → Отправить → Сжатая ZIP-папка

5. Назовите: project.zip


═══════════════════════════════════════════════════════════════════════
  ШАГ 3: ЗАГРУЖАЕМ НА GITHUB (2 минуты)
═══════════════════════════════════════════════════════════════════════

1. На странице вашего пустого репозитория найдите:
   "uploading an existing file"
   
   Или просто нажмите:
   "Add file" → "Upload files"

2. Перетащите project.zip в окно браузера
   ИЛИ
   Нажмите "choose your files" и выберите project.zip

3. Внизу напишите:
   Commit message: Initial commit

4. Нажмите: Commit changes

5. ПОДОЖДИТЕ пока файл загрузится (может быть 1-2 минуты)


═══════════════════════════════════════════════════════════════════════
  ШАГ 4: РАСПАКОВЫВАЕМ НА GITHUB (1 минута)
═══════════════════════════════════════════════════════════════════════

СТОП! GitHub не распакует ZIP автоматически!

Надо загрузить файлы БЕЗ архива:

1. Вернитесь на главную страницу репозитория

2. Нажмите "Add file" → "Upload files"

3. Откройте project.zip на компьютере (распакуйте)

4. Выделите ВСЕ папки и файлы ВНУТРИ:
   - Client (папка)
   - Server (папка)
   - Shared (папка)
   - BUILD_EXE.bat
   - MASTER_SETUP.bat
   - И все остальные файлы

5. Перетащите ВСЁ в окно GitHub

6. Commit message: Add all files

7. Commit changes

8. Подождите загрузки (2-3 минуты)


═══════════════════════════════════════════════════════════════════════
  АЛЬТЕРНАТИВА: ЗАГРУЖАЕМ ПО ЧАСТЯМ (если файлов много)
═══════════════════════════════════════════════════════════════════════

Если GitHub ругается на количество файлов:

1. Загрузите сначала ТОЛЬКО:
   - Dockerfile
   - railway.json
   - Server (папка целиком)
   - Shared (папка целиком)

2. Commit changes

3. Потом Add file → Upload files снова:
   - Client (папка целиком)

4. Commit changes

5. Потом все .bat и .md файлы

6. Commit changes


═══════════════════════════════════════════════════════════════════════
  ШАГ 5: ПРОВЕРКА (10 секунд)
═══════════════════════════════════════════════════════════════════════

Ваш репозиторий должен выглядеть так:

hardcore-messenger/
├── Client/
├── Server/
├── Shared/
├── BUILD_EXE.bat
├── MASTER_SETUP.bat
├── Dockerfile
├── railway.json
└── остальные файлы

Если так - ОТЛИЧНО! Переходим к Railway!


═══════════════════════════════════════════════════════════════════════
  ШАГ 6: RAILWAY ДЕПЛОЙ (5 минут)
═══════════════════════════════════════════════════════════════════════

1. Откройте: https://railway.app

2. Нажмите: Start a New Project

3. Выберите: Deploy from GitHub repo

4. Найдите: hardcore-messenger

5. Нажмите на него

6. Railway начнёт деплой! Подождите 2-3 минуты

7. Когда увидите зелёную галочку - сервер запущен!

8. Переходите к Шагу 7


═══════════════════════════════════════════════════════════════════════
  ШАГ 7: ПОЛУЧАЕМ URL СЕРВЕРА (1 минута)
═══════════════════════════════════════════════════════════════════════

1. В Railway нажмите на ваш проект

2. Слева найдите: Settings

3. Найдите раздел: Networking

4. Нажмите: Generate Domain

5. Вы получите URL типа:
   hardcore-messenger-production.up.railway.app

6. СКОПИРУЙТЕ ЕГО!


═══════════════════════════════════════════════════════════════════════
  ШАГ 8: СОЗДАЁМ EXE ЛОКАЛЬНО (5 минут)
═══════════════════════════════════════════════════════════════════════

1. Вернитесь в папку проекта на компьютере

2. Запустите: BUILD_EXE.bat
   (Правой кнопкой → Запуск от имени администратора)

3. Дождитесь завершения

4. Откройте папку: Distribution\

5. Откройте файл: ClientConfig.txt

6. Замените:
   ws://YOUR_RAILWAY_URL_HERE
   
   На:
   ws://hardcore-messenger-production.up.railway.app
   (ваш URL из Шага 7)

7. Сохраните (Ctrl+S)


═══════════════════════════════════════════════════════════════════════
  ШАГ 9: ТЕСТИРУЕМ! (1 минута)
═══════════════════════════════════════════════════════════════════════

1. Откройте: Distribution\HardcoreMessenger.exe

2. Если Windows ругается:
   Подробнее → Выполнить в любом случае

3. Введите имя

4. Подключиться

5. РАБОТАЕТ! 🎉


═══════════════════════════════════════════════════════════════════════
  ШАГ 10: РАСПРОСТРАНЕНИЕ
═══════════════════════════════════════════════════════════════════════

1. Выделите ВСЁ в папке Distribution\

2. Правой кнопкой → Отправить → Сжатая ZIP-папка

3. Назовите: HardcoreMessenger_v2.5.zip

4. Загрузите на:
   - Google Drive
   - Яндекс.Диск
   - GitHub Releases (в вашем репозитории)

5. Делитесь ссылкой!

Q: Можно без GitHub?
A: Нет, Railway требует GitHub репозиторий

Q: Можно без Railway?
A: Да, но тогда сервер только локально (не в интернете)

Q: BUILD_EXE.bat не работает?
A: Установите .NET 8.0 SDK и перезагрузите компьютер

Q: Клиент не подключается?
A: Проверьте ClientConfig.txt - правильный ли URL?
   Проверьте Railway - сервер запущен?
